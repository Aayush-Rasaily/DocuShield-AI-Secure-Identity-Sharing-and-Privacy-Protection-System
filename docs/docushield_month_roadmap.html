Week 1 · Mar 16–22 — AI Pipeline Foundation 10 tasks
Download Aadhaar dataset from Roboflow
AI
Universe: akash-k-p-gs9iu/aadhaar-card-details-extraction — 2,900+ images, 5 classes. Split 80/20 train/val.
Create dataset YAML and verify class labels
AI
Classes: AadhaarNumber(0), DOB(1), Gender(2), Name(3), Address(4). Validate annotation format.
Run baseline YOLOv8-small training (100 epochs)
AI
model.train(data="aadhaar.yaml", epochs=100, imgsz=640, batch=16, optimizer="Adam", lr0=0.001)
Add data augmentation: glare, blur, skew, crop
AI
Handles laminated card reflections. Target: improve Address class from baseline ~83% mAP-50.
Implement preprocessing pipeline in OpenCV
AI
Steps: grayscale → Gaussian blur (5×5) → Canny edge → perspective transform → normalize [0,1]
Integrate Tesseract OCR (lang: hin+eng)
AI
PSM 6 for Aadhaar number regions, PSM 11 for address. Crop bounding box output → feed to Tesseract.
Build regex post-processor for each field class
AI
Aadhaar: r"\d{4}\s\d{4}\s\d{4}", DOB: r"\d{2}/\d{2}/\d{4}", Address: RapidFuzz fuzzy match + PIN regex
Set up FastAPI project scaffold + Docker Compose
BackendInfra
Services: fastapi, postgres:15, redis:7, minio. Alembic migrations. Health check endpoints.
Set up Expo Router project with route groups
Mobile
Groups: (auth)/ for login/register, (vault)/ for protected screens. Redirect unauthenticated to /sign-in.
Write unit tests for preprocessing + OCR pipeline
Test
20 test images covering: laminated, blurry, Hindi+English, e-Aadhaar screenshot. Target: ≥ 90% field extraction.
Week 2 · Mar 23–29 — Backend Core + Model Tuning 9 tasks
Export best.pt to ONNX and validate inference
AI
model.export(format="onnx", dynamic=False, imgsz=640). Run onnxruntime on 10 images, verify bboxes match .pt output.
Build ELA forgery detection module
AI
Save at quality=90, compute pixel diff. ELA score > 15 → Suspicious. Pair with CNN binary classifier.
Build QR code payload validator
AI
Decode QR with pyzbar → extract Aadhaar number → compare with OCR output. Mismatch = Likely Forged.
Implement JWT auth: register, login, refresh
Backend
Access token 15min (HS256), refresh token 7d. bcrypt password hash. POST /auth/register, /auth/login, /auth/refresh.
POST /documents/upload — file + AI pipeline
Backend
Accept UploadFile. Run forgery check → YOLOv8 → OCR → return field detections. BackgroundTasks for heavy ops. 8s budget.
POST /documents/{id}/mask — apply masking config
Backend
Accept masking JSON per field. Return masked image URL (pre-signed MinIO, 1hr TTL).
SQLAlchemy models: User, Document, VaultItem, ShareToken
Backend
Alembic migration. Document stores encrypted_path, masking_presets (JSONB), forgery_status. ShareToken stores bcrypt_hash + expires_at.
Auth screens: Login + Register in Expo Router
Mobile
useSession hook with JWT in expo-secure-store. Protected (vault) layout redirects to /sign-in if !session. Biometric via expo-local-authentication.
API integration tests: auth + upload + mask
Test
pytest + httpx async client. Test happy path and error cases: bad image, expired token, duplicate upload.
Week 3 · Mar 30–Apr 6 — Mobile UI + Sharing Engine 9 tasks
Camera capture screen with image quality check
Mobile
expo-image-picker: launchCameraAsync. Client-side blur score (Laplacian variance < 100 → re-capture prompt).
AI processing loader screen
Mobile
Steps: "Checking authenticity…" → "Detecting fields…" → "Ready to customize". Forgery badge shown on result.
Masking toggle UI with live document preview
Mobile
Per-field toggles (Show/Partial/Mask). Preview re-renders on each toggle. "Save preset" button.
Vault CRUD: encrypt + store in MinIO
Backend
AES-256-GCM. Key = PBKDF2(password, salt, 100000 iters). Server never holds plaintext key.
Share token engine: Redis TTL + link + QR
Backend
Token key: share:{uuid4}. TTL: 3600/21600/86400/604800s. one-time flag sets max_access=1. QR = qrcode.make(link).
Masked PDF generator with watermark
Backend
ReportLab: embed masked image. Diagonal watermark: "COPY — NOT FOR REUSE". Footer: "Shared via DocuShield AI | {timestamp}".
Share screen: link + QR fullscreen + PDF button
Mobile
3 tabs: Link (copy + expiry timer), QR (fullscreen), PDF (single download). Access count + Revoke button.
Vault list screen + document detail + share log
Mobile
FlatList of saved documents. Detail: masking presets, share history (time, access count, IP). Revoke from here.
GET /share/{token} — public share endpoint
Backend
No auth. Validate Redis token → increment access_count → 410 Gone if one-time used. Log IP + timestamp.
